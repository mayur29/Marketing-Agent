# Marketing Agents

This project defines a hospitality-specific system of AI marketing agents for DISH that can handle a large share of recurring marketing work while keeping humans in control of strategy, approvals, and brand risk.

The aim is to build a practical multi-agent marketing operating system for restaurant and hospitality growth, not a collection of disconnected prompts.

## What This Repo Contains

- `MARKETING_AGENTS.md` for the system-level vision and architecture
- `PRODUCT_PLAYBOOK.md` for what stays constant and what changes by DISH product
- `RUNBOOK.md` for human operation of the system
- `RUN_WORKFLOW.md` for the end-to-end V1 operating flow
- `agents/` for role-specific agent definitions
- `knowledge/` for shared business and market context
- `outputs/` for campaign artifacts generated by the workflow
- `scripts/` for lightweight automation utilities
- `templates/` for reusable output formats

## V1 Goal

Build a small set of high-leverage agents that can support an end-to-end hospitality marketing workflow for DISH:

1. understand business goals
2. research the hospitality market
3. shape positioning and messaging for restaurant operators
4. plan campaigns
5. draft marketing assets
6. analyze results and recommend next actions
7. localize and translate (DE/EN focus)

## Operating Model

The agents should work as a coordinated system:

1. The CMO Agent turns company goals into priorities
2. The Market Research Agent updates competitive and audience context
3. The Messaging and Positioning Agent defines the core angle
4. The Campaign Strategist Agent creates the campaign plan
5. The Copywriter Agent creates channel-ready drafts
6. The Analytics and Optimization Agent closes the loop with learnings

## Guardrails

- humans approve publishing and budget decisions
- agents must use approved product facts and claims
- all output should be traceable to source context
- campaigns must stay grounded in approved DISH product, pricing, and compliance context
- **Language Policy**: Primary language is determined by the CMO Agent per campaign; all downstream handoffs must preserve locale and tone guidance, including German `Sie` vs `Du`
- brand consistency matters more than content volume
- agents should draft, recommend, and summarize before they automate
