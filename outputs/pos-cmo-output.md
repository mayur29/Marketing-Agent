# CMO Output

## Campaign

DISH POS DE Conversion Push

## Strategic Priority

Generate qualified demo requests from German restaurant operators for DISH POS by positioning it as a practical way to improve checkout reliability, simplify operations, and support German fiscalization requirements.

## Why This Matters Now

- checkout speed and operational reliability directly affect service quality
- fragmented tools create complexity for hospitality teams
- German operators care about compliance and need practical systems that fit daily operations

## Selected Audience

- independent restaurants in Germany
- multi-location hospitality operators needing consistent checkout workflows
- operators frustrated by slow checkout, fragmented POS tools, or compliance complexity

## Selected Offer

DISH POS, an all-in-one cloud-based point of sale system for hospitality with offline mode, inventory management, and German fiscalization support.

## Market and Language

- market: Germany
- primary language: German
- tone register: DE `Sie`

## Recommended Campaign Direction

Lead with an operations-first value proposition:

- faster and more reliable checkout
- one POS workflow built for hospitality
- support for German fiscalization requirements

The campaign should stay practical, trustworthy, and operationally credible.

## Agent Assignments

- Market Research Agent
  Validate operator pain points around checkout speed, system complexity, and compliance concerns in Germany.
- Messaging and Positioning Agent
  Create a proof-backed message hierarchy for German restaurant operators evaluating POS systems.
- Campaign Strategist Agent
  Build a focused conversion campaign once messaging is approved.

## Risks and Assumptions

- compliance messaging can become too technical if not translated into operator value
- operators may care more about service speed than software architecture
- claims around reliability and performance should stay tied to approved product context

## Escalations

- human review required before making hard performance, uptime, or compliance claims beyond approved product proof
- human review required if implementation promises exceed approved DISH context
