## Positioning Statement

For restaurant operators in Germany who need a faster, more reliable, and more manageable checkout workflow, DISH POS provides an all-in-one hospitality point-of-sale system with practical support for daily operations and German fiscalization requirements.

## Core Value Proposition

Run checkout and in-restaurant operations with a POS system built for hospitality, with stronger reliability, simpler workflows, and support for German compliance needs.

## Key Messages

- improve checkout flow with a hospitality-focused POS system
- reduce operational complexity with one connected workflow
- stay better prepared for day-to-day service interruptions with offline capability
- support German fiscalization requirements with a system built for the market

## Proof Points

- all-in-one cloud-based POS for hospitality
- offline mode
- fiscalization compliance support for Germany
- inventory management

## Objections to Address

- concern about switching systems and staff training
- uncertainty about implementation effort
- doubt about whether the system will fit the restaurant's daily service reality

## Tone Guidance

Practical, reliable, operationally grounded, and confidence-building. Avoid overly technical software language or inflated performance claims.

## Recommended Campaign Angles

- faster checkout, simpler operations
- hospitality POS built for Germany
- more control at the point of sale

## What To Emphasize

- checkout reliability
- operational simplicity
- hospitality fit
- compliance reassurance

## What To Avoid

- unsupported uptime guarantees
- technical jargon without operator value
- exaggerated claims about implementation ease
